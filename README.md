# Архив — персональный трекер медиа

Единый каталог для фильмов, сериалов, аниме, книг, манги и игр — с обложками, статистикой, картой стран и импортом из Steam/Shikimori.

## Запуск

Просто открой `index.html` в браузере (двойной клик по файлу). Работать нужно именно как с открытым локально файлом, а не через встроенные превью — иначе не будут работать сетевые функции (поиск по базам, импорт).

## Возможности

- Единый каталог 6 типов медиа с разными полями под каждый тип
- Поиск/автозаполнение по открытым базам (TMDB*, iTunes, TVmaze, Jikan, Open Library, RAWG*, CheapShark, RetroAchievements)
- Импорт коллекций:
  - **Игры** — Steam (Web API), RetroAchievements
  - **Фильмы/сериалы** — Letterboxd (CSV), IMDb (CSV) + дозаполнение постеров/описаний из TMDB*
  - **Книги** — Goodreads (CSV), обложки по ISBN из Open Library
  - **Аниме/манга** — Shikimori (по нику), AniList (по нику), MyAnimeList (XML-экспорт)
- Импорт/экспорт CSV и JSON (универсальный, с сопоставлением колонок)

\* TMDB и RAWG требуют бесплатный API-ключ — вводится во вкладках «Кино» и «Steam» окна импорта; без ключа работают запасные базы (iTunes/TVmaze/CheapShark).
- Статистика: тепловая карта активности, диаграммы, карта мира по странам
- Тёмная/светлая тема, режим "стена постеров"
- Данные хранятся локально в браузере (window.storage)
- Опционально: регистрация/вход и синхронизация архива между устройствами (Supabase)

## Аккаунты и синхронизация

По умолчанию всё хранится локально в браузере. Чтобы включить вход и облачную синхронизацию:

1. Создай бесплатный проект на [supabase.com](https://supabase.com)
2. В SQL Editor выполни скрипт из `db/supabase-setup.sql` (таблица архива + row-level security)
3. Скопируй Project URL и anon-ключ (Settings → API) в `js/config.js`

После этого в шапке появится «Войти». Регистрация — email + пароль. Архив синхронизируется автоматически при каждом изменении; конфликты сливаются по принципу «новее — побеждает», удаления переносятся между устройствами через тумбстоуны. anon-ключ публичный по дизайну (данные защищает RLS), его можно коммитить. Ключи API (Steam, TMDB и т.д.) не синхронизируются и остаются на устройстве.

## Автономное обогащение и локальные обложки (скрипт)

Для 100% автономной работы без сторонних API и хостингов постеров (которые могут блокировать горячие ссылки) предусмотрен Node.js-скрипт `scripts/enrich.js`.

Скрипт автоматически находит описания, разработчиков, режиссеров, авторов и жанры через официальные API (TMDB, RAWG, Shikimori, Google Books) и **скачивает обложки локально** в папку `data/covers/`.

### Инструкция по запуску:
1. Зайдите в приложение в браузере и нажмите **Импорт / Экспорт** → **Экспорт JSON**.
2. Переместите скачанный файл `archive-export.json` в корневую папку этого проекта.
3. Создайте файл `.env` в корневой папке и добавьте ваши API-ключи:
   ```env
   TMDB_API_KEY=твой_ключ_tmdb
   RAWG_API_KEY=твой_ключ_rawg
   ```
4. Убедитесь, что у вас установлен Node.js, и запустите скрипт в терминале:
   ```bash
   node scripts/enrich.js
   ```
5. Скрипт скачает все обложки в `data/covers/` и обновит файл `archive-export.json`, заменив ссылки на локальные пути.
6. Откройте приложение, нажмите **Импорт / Экспорт** → вкладка **Файл** и загрузите обновленный `archive-export.json`.

Все локальные обложки теперь будут загружаться прямо с вашего компьютера/репозитория!

## Стек

Без сборки, vanilla JS + CSS, разбито по файлам:

- `index.html` — разметка
- `css/style.css` — стили
- `js/app.js` — логика приложения
- `js/worldmap.js` — данные SVG-карты мира (сжатая карта из открытого пакета `@svg-maps/world`, MIT)
